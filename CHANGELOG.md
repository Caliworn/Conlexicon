# Changelog

本文件记录 Conlexicon 的重要功能、行为与兼容性变更。

## New

### 新增

- 新增 repository/model 轻量检查脚本，使用临时数据目录验证词典规范化、导入解析、JSON repository 的创建、更新、导入覆盖、导出、删除、偏好保存和错误状态。

### 改进

- 阶段 B 开始建立数据访问层边界：新增 `JsonDictionaryRepository`、词典模型规范化模块、API 路由模块、HTTP 工具模块和静态文件服务模块，将词典索引、词典 JSON 快照、导入、导出、创建、激活、删除、偏好设置、导入解析、数据规范化、API 分发和静态文件响应从 `server.js` 中抽离；`server.js` 现在只负责组装 repository、路由、静态服务并启动 HTTP 服务器，为后续索引、分页和 SQLite 评估保留兼容入口。
- 词典实体 ID 唯一性检查扩展覆盖释义、形态表和 IPA 映射规则；前端保存/导入提示与后端 repository 校验现在会一起识别这些对象的重复 ID，缺失 ID 的规范化补齐也会对这些对象和新词典 ID 执行静默防撞。
- 导入 JSON 时如遇词典 ID 格式无效，会提示是否作为新词典导入并重新生成 ID；未确认时仍拒绝无效 ID，避免静默改写来源数据。

## 2026-07-02

### 新增

- 中文品牌名更新为“构典”，并同步 README 与界面标题。
- 新词典的自动 IPA 设置默认开启“单音节词不标重音”。
- 新增 `Ctrl`/`Cmd` + `Enter` 新建词条快捷键；焦点位于输入框、文本框、下拉框或其他可编辑区域时不会触发，并沿用未保存编辑的“保存 / 放弃 / 取消”确认流程。
- README 新增快捷键说明，记录保存快捷键与新建词条快捷键。
- 词条详情中的未记载来源现在可点击创建为空来源词条，新建草稿会自动以该来源形式填充词形。
- 新增本地压力测试词典生成脚本，可在 `data/` 下生成使用规范类型前缀 UUID 的 1000 条随机词条测试词典。
- 其他设置新增“词条列表中显示原始标签”开关，可让词条列表标签忽略显示替换并直接显示原始标签。
- 其他设置新增词条列表标签显示上限，可配置 2–10 个标签 chip，默认为 3，并将省略号计入上限。

### 改进

- 新增 `AGENTS.md` 作为长期协作与工程约束文档，记录任务类型与授权边界、测试端口管理、验证命令、changelog 提交流程、README 更新要求、数据不变量、持久化边界和 UI 验收检查项；`NEXT_IMPLEMENTATION_HANDOFF.md` 改为引用该文件并专注阶段性交接；README 补充响应式应用外壳说明。
- 逐步淘汰原生 title 提示：导航与 Gloss 样式等应用内 tooltip 控件改用 `aria-label` 翻译键，并移除运行时清理导航 title 的旧逻辑。
- 删除旧的 `data-i18n-title` 本地化兼容逻辑，后续提示文案统一通过 `aria-label` 与应用内 tooltip 承载。
- 应用启动时使用统一的 `app-booting` 水合阶段隐藏主界面壳，等待后端状态、语言、响应式导航和首轮渲染完成后再显示，减少刷新时默认标题、按钮、控制栏和导航状态短暂闪烁。
- 启动阶段会读取本地主题缓存以提前应用暗黑背景；真实主题设置仍以 `data/index.json` 为准，并在后端状态返回或主题切换后同步缓存。
- 释义编辑中的“例句”和“释义备注”改为按需添加：默认只显示释义输入框，有既有内容时自动展开，否则通过“添加例句”和“添加释义备注”按钮手动创建；完整编辑和局部编辑保持一致。
- “添加释义”“添加例句”和“添加释义备注”统一使用绿色描边的正向添加按钮样式。
- 词典管理中的 JSON 导出入口移至词典列表卡片，每个词典都可直接导出；当前词典在列表中以绿色“当前”状态按钮标记，切换当前词典的入口也统一保留在列表卡片内。
- 移动端新增紧凑应用栏，并将工具导航与词条列表接入抽屉式打开/关闭骨架；选择模块、选择词条或新建词条成功后会自动回到主内容区。
- 触发标签、词性、高级筛选或数据分析“当前搜索命中”等词条结果入口时，会自动显示词条列表：移动端打开词条列表抽屉，桌面端自动展开已收起的词条列表。

### 修复

- 修复移动端抽屉打开时，保存/放弃/取消等通用确认弹窗会被抽屉遮挡的问题；确认弹窗层级现在高于工具导航与词条列表抽屉。
- 修复词条列表隐藏或抽屉关闭时触发词条跳转/保存后，自动滚动可能因隐藏状态测量不可靠而失效的问题；现在会记录待聚焦词条，并在列表重新显示后滚动到当前词条。
- 修复极窄宽度下词条详情头部操作按钮换行后仍被左侧标题与标签块撑开，导致按钮与标签之间出现大块空白的问题。
- 修复极窄移动端下内容重排可能挤到顶部应用栏上方或被顶部应用栏覆盖的问题；移动端应用栏改为占位的 sticky 首行。
- 修复中等宽度下刷新时工具导航先按展开状态渲染再自动收起、词条列表跟随展开态位置短暂移动的问题；启动/稳定阶段会暂时禁用关键布局过渡，并在显示前先完成词条列表测量。
- 修复刷新或开屏时词条列表在 fixed 布局变量尚未测量完成前短暂出现在左上角的问题；词条列表只有在布局测量 ready 后才进入 fixed 定位。
- 修复新建词条保存后、完整编辑或局部编辑改名导致排序位置变化后，词条列表不会自动滚动到保存后词条位置的问题。

- 标签排序、质量检查高级筛选、自动形态学语法和 IPA 音节划分说明统一改用同一套信息弹窗，并将说明按钮提示统一为应用内 tooltip。

- 词条列表中可见标签 chip 的应用内 tooltip 改为统一显示“原始标签”和已配置的“显示替换”，避免依赖不稳定的视觉截断检测；省略号 chip 仍只显示被隐藏标签列表。
- 其他设置中的 Gloss 样式控件改用应用内 tooltip，不再触发浏览器原生 title 提示。

- 新增词典导入 ID 冲突确认，可选择覆盖现有词典或取消导入。

### 改进

- 页面切换时的局部编辑和完整编辑行为改为“保存”“放弃更改”或“弹窗提示”三种模式。
- 词根模式跳转到未展开的衍生词时会自动展开目标词根，并正确定位多来源词条的对应实例。
- 语料单元编辑内容现在可以即时更新渲染预览，不必等待保存完成。
- 自动保存不再退出当前语料编辑状态，也不会清除当前草稿和撤销历史。
- 数据分析与其他设置卡片改为响应式瀑布流布局，长卡片不再撑开同一网格行。
- 自动 IPA、自动形态学和其他设置移除不必要的最外层大卡片，使子区域边界更清晰。
- 其他设置改进宽屏空间利用，并调整渲染对象、标签替换等长文本输入区域。
- README 补充语料库、Gloss 样式、保存策略、导入保护与全局语言设置说明。
- IPA 映射改为严格按照列表从上到下匹配；规则始终读取原始词形，生成结果不会被后续规则再次读取，较早规则消耗的字符不会再参与后续匹配。
- 改进中间宽度与窄屏下的通用布局收缩和换行；词条详情、设置、分析等卡片不再被内部 Grid、Flex 或长文本撑出边界。
- 形态表格与对齐 Gloss 在空间不足时保持卡片内部局部横向滚动，不再引起页面级横向滚动。

- 修复语料库为空时仍存在额外滚动空间的问题。
- 修复 Gloss 多行对齐未生效、`\ft` 参与对齐以及对齐后字号异常缩小的问题。
