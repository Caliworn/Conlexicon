# Changelog

本文件记录 Conlexicon 的重要功能、行为与兼容性变更。

## New

### 新增

- SQLite repository 骨架新增 `patchEntries()`，补齐批量词条 patch 语义，支持 `tags`、`pronunciation` 字段更新和同步 settings 合并。
- SQLite repository 骨架新增 `queryRootGroups()`，支持词根分组的搜索、排序、分页和 full/summary 返回语义；无搜索场景已使用 SQL projection 构建词根分组，搜索型请求仍保留完整共享语义回退。
- SQLite repository 骨架新增 `getEntryRelations()`，按现有响应格式返回来源匹配、衍生条目和同根组；来源和衍生查询已使用 SQL projection。
- SQLite repository 骨架新增 `getEntryFacets()`，支持词性 facets、标签频率和无词性计数的现有语义；当前已使用 SQL 聚合。
- SQLite repository 骨架的 `queryEntries()` 新增搜索、词性/标签/来源派生筛选、排序、分页和 summary/full 返回语义；结构化筛选、排序和分页已使用 SQL，下阶段再处理全文/模糊/动态形态搜索索引。
- SQLite repository 骨架新增 metadata、settings、docs、corpus、morphology 和 IPA 模块级保存方法；当前仍通过 module blob/projection 重写保证语义，尚未接入主服务。
- SQLite repository 骨架新增 skeleton 级词条 CRUD，支持无筛选 `queryEntries()`、`getEntry()`、`saveEntry()` 和 `deleteEntry()`，尚未接入主服务。
- SQLite repository 骨架新增最小词典生命周期方法，覆盖创建、导入、导出、激活、删除、偏好保存和 state 读取，仍未接入主服务。
- SQLite repository 骨架新增 JSON ↔ SQLite 最小无损往返：导入时写入完整词条 JSON、模块 blob 和核心查询 projection，导出时还原完整词典 JSON。
- 新增未接入主流程的 `SqliteDictionaryRepository` 骨架和 SQLite schema 初始化检查脚本，先验证 `.sqlite` 文件、schema migrations、核心词条表、模块 blob 表和第一批索引可在临时目录创建。
- 新增 `SQLITE_BACKEND_PLAN.md`，明确 SQLite 作为后续主存储的草案、schema 初稿、迁移顺序，并反推启动、词条查询、facets、词根关系、数据分析、质量检查、保存返回值和诊断修复等关键 API 契约。
- 新增 `SQLITE_MIGRATION_PLAN.md`，明确 SQLite 作为未来 canonical storage、JSON 退位为 legacy import/export profile，并规划转换服务、目录迁移脚本、导出 profile、备份策略和迁移报告。
- 新增 `dictionary-conversion-service` 骨架，集中 legacy JSON 导入解析和导出 profile 校验；`/api/import` 与 `/api/export` 已接入该服务，默认行为保持兼容完整 JSON。
- 新增 `scripts/migrate-json-data-to-sqlite.js` 迁移脚本，可将显式指定的旧 JSON data directory 转换到新的 SQLite data directory；脚本拒绝非空目标目录和嵌套源/目标目录，并输出 JSON migration report。
- 新增 `scripts/check-sqlite-migration.js`，使用临时目录验证 JSON 到 SQLite 迁移不会修改源目录，并能保留 active dictionary、界面偏好和词典内容。
- Repository 契约测试新增 corpus blob ID 防撞覆盖，确认未 SQL 化的语料 block/layer/unit ID 仍会与词条、释义等实体共享全局 ID 命名空间。
- 新增 repository/model 轻量检查脚本，使用临时数据目录验证词典规范化、导入解析、JSON repository 的创建、更新、导入覆盖、导出、删除、偏好保存和错误状态。
- 阶段 B2 开始接入词条级 API：新增词条列表、单条读取、单条保存和删除端点，并在 repository 检查脚本中覆盖这些端点。
- 新增设置、文档、语料、自动形态学、自动 IPA 与批量词条 patch 的模块级 API，为继续拆分完整词典保存路径做准备。
- 新增 `API_CONTRACT.md`，记录当前本地 HTTP API、完整快照兼容层、增量保存端点、实体 ID 校验范围和结构化错误码；`AGENTS.md` 增加同步维护 API 契约的长期规则。
- `API_CONTRACT.md` 新增阶段 B3 读取 API 草案，覆盖轻量启动状态、词条查询、facets、词源关系、语料读取和数据分析按需 summary 的后续契约方向。
- `API_CONTRACT.md` 补充共享查询索引、数据分析 widget query、planner、filter descriptor、诊断修复边界和 SQLite 兼容设计，作为后续数据分析 API 化与通用查询层抽取的草案。
- 新增第一批读取侧 API：词条列表查询参数、词条 facets 和词源/衍生/同根关系端点，并在 repository 检查脚本中覆盖。

### 改进

- SQLite 开发期 schema 新增 `entries.etymology_description` 投影列和 `entry_morphology_tables` 表，为从 SQL projection 重建完整词条和未来一词条多形态表实例做准备；当前 SQL 开发中间态不做兼容迁移，测试 SQLite 目录需从 JSON 重新生成。
- SQLite repository 移除 `entries.entry_json` 存储字段；词条写入、单条读取、列表读取、词源关系、词根分组和导出快照现在都以 SQL projection 表为主结构，JSON 导出由 SQL 表组装生成。
- SQLite 检查脚本拆分职责：新增 schema/projection 检查、lifecycle smoke 和共享 SQLite 检查工具，`check-sqlite-repository.js` 改为薄聚合入口，完整行为一致性继续由 `check-sqlite-contract.js` 负责。
- `server.js` 新增 `CONLEXICON_REPOSITORY=json|sqlite` feature flag，默认继续使用 JSON repository；显式设置为 `sqlite` 时会启动实验性 SQLite repository，README、API 契约和 SQLite 计划同步记录该运行方式。
- 前端启动流程改为先读取轻量 `/api/state`，再按需通过 `/api/dictionaries/:id` 加载当前词典完整快照；已加载且 `updatedAt` 未变化的词典会复用本地 snapshot，避免轻量 state 刷新时把前端词典内容覆盖为空。
- API 新增 `GET /api/dictionaries/:id` 完整词典快照读取端点，供启动、切换词典和兼容层按需加载使用；`API_CONTRACT.md` 同步记录轻量 state 与 active dictionary snapshot 的边界。
- 词典管理列表和删除确认的词条/词根统计优先使用 `summary.entryCount/rootCount`，避免轻量 state 下为 metadata-only 词典显示 0 或触发完整词典统计。
- 保存 API 响应继续收窄：metadata、settings、docs、corpus、morphology、IPA 设置、autosave 和批量词条 patch 现在返回局部 payload；前端保存后直接合并局部响应，不再为普通保存重新拉取 `/api/state` 和 active dictionary snapshot。
- `SQLITE_BACKEND_PLAN.md` 新增 SQLite repository 当前状态审计表，区分已 SQL 增量写入、已 SQL 查询下推、仍走完整快照/JS 全量逻辑和主服务尚未接入的部分。
- SQLite repository 的 metadata、settings、IPA、docs、morphology 和 corpus 模块保存改为 SQL 级写入，直接更新 `dictionary_meta` 或对应 `module_blobs`，并保留 IPA、形态和语料的局部实体 ID 校验。
- SQLite repository 的 `saveEntry()`、`deleteEntry()` 和 `patchEntries()` 改为真正 SQL 增量写入：只更新目标词条、释义 projection、标签 projection、来源 projection、相关 settings blob 和词典更新时间，不再为这些操作重写整库 projection。
- SQLite repository 的结构化读取开始下推到 SQL：无全文搜索的 `queryEntries()` 直接使用 `entries`、`entry_tags` 和 `entry_sources` projection 表完成词性、标签、来源、derivedFrom、排序和分页；`getEntryFacets()` 改为 SQL 聚合；`getEntryRelations()` 改为使用 SQL 来源索引查询来源和衍生条目。
- SQLite repository 的无搜索 `queryRootGroups()` 改为使用 SQL projection rows 构建词根分组、排序和分页，不再为普通词根模式入口导出完整词典；带全文、fuzzy 或动态形态搜索的 root groups 暂时继续回退共享 JS 语义。
- SQLite repository 的 `readState()` / `listDictionaries()` 改为只读取 `dictionary_meta` 并返回轻量词典 metadata 列表，附带即时 SQL 计算的 `summary.entryCount/rootCount`，避免启动索引阶段导出每个完整词典且不把派生统计写入 metadata；`API_CONTRACT.md` 同步记录正式接主服务前还需要前端按需加载 active dictionary。
- SQLite repository 的共享契约检查推进到完整契约，覆盖模块保存、批量词条 patch、偏好、导入覆盖、删除和错误状态。
- SQLite repository 的共享契约检查从 `entryCrud` 推进到 `readApi`，确认基础读取 API、facets、词汇关系、词根分组和读取一致性矩阵已与 JSON repository 当前语义对齐。
- Repository 契约测试 runner 新增早停阶段；SQLite repository 现在可复用同一套契约检查并通过到 `entryCrud` 阶段。
- Repository 检查脚本拆分为可复用契约测试 runner 和 JSON repository 入口，为后续 SQLite repository 复用同一套 API/保存/导入导出语义检查做准备。
- 建立最小共享 query 层，先接管数据分析中的词根/衍生关系统计和词根家族查询，为后续 API 化与共享索引查询打基础。
- 阶段 B 开始建立数据访问层边界：新增 `JsonDictionaryRepository`、词典模型规范化模块、API 路由模块、HTTP 工具模块和静态文件服务模块，将词典索引、词典 JSON 快照、导入、导出、创建、激活、删除、偏好设置、导入解析、数据规范化、API 分发和静态文件响应从 `server.js` 中抽离；`server.js` 现在只负责组装 repository、路由、静态服务并启动 HTTP 服务器，为后续索引、分页和 SQLite 评估保留兼容入口。
- 词典实体 ID 唯一性检查扩展覆盖释义、形态表和 IPA 映射规则；前端保存/导入提示与后端 repository 校验现在会一起识别这些对象的重复 ID，缺失 ID 的规范化补齐也会对这些对象和新词典 ID 执行静默防撞。
- 导入 JSON 时如遇词典 ID 格式无效，会提示是否作为新词典导入并重新生成 ID；未确认时仍拒绝无效 ID，避免静默改写来源数据。
- 新建、完整编辑、局部编辑和删除词条的前端保存路径开始使用词条级 API，完整词典快照端点暂时保留为兼容层。
- 其他设置、语言文档、语料库、自动形态学、自动 IPA、自动整理标签顺序和批量 IPA 生成改用模块级或批量词条 API；批量 IPA 生成检测到未保存 IPA 设置时会要求先保存设置再继续。
- 前端移除旧的完整词典保存 helper，普通运行期保存路径不再直接调用完整词典 PUT；后端完整快照端点仍保留为兼容层。
- 词典管理的名称、语言和描述保存改用词典元数据 API；页面卸载时的文档/语料自动保存兜底也统一走 autosave 入口。
- 自动整理标签顺序采用与批量 IPA 相同的严格逻辑：检测到其他设置未保存时，会要求先保存设置再继续应用排序。
- 批量词条 patch 增加字段白名单，目前仅允许更新 `tags` 和 `pronunciation`，并拒绝错误类型，避免批量端点绕过词条级校验修改嵌套结构或实体 ID。
- 增量保存改为局部实体 ID 校验：设置、文档、批量非 ID 词条更新等无关保存不再被历史重复 ID 阻断；词条、IPA、形态和语料等会改动实体 ID 的模块仍会检查本次保存范围内的冲突，导入和完整快照保存继续执行全量检查。
- 自动保存端点改为分发到文档和语料模块级保存，不再通过完整词典 patch 触发全量 ID 校验。
- 后端 API 错误改为返回稳定错误码；前端保存、导入、词典切换、偏好保存等失败时会读取错误码并显示本地化短提示，控制台仍保留原始技术错误。
- 词条列表普通模式新增读取 API 预备层：搜索、词性筛选和排序会调用后端词条查询 API，并在结果与本地筛选完全一致时使用 API 返回的顺序；不一致或请求失败时自动回退本地筛选。
- 词性筛选选项新增 facets API 接线：前端正常路径读取后端 facets，不再每次本地统计完整词性集合做一致性校验；本地统计逻辑暂时保留为 API 不可用时的轻量兜底。
- 读取 API 检查脚本新增一致性矩阵，覆盖普通搜索、标签显示替换、手动/默认词性识别、无词性、多词性、排序稳定性、分页和 facets，防止后端读取语义偏离前端旧行为。
- 词条读取 API 的词条排序和 facets 词性排序调整为与前端旧逻辑一致，减少读取预备层不必要回退。
- 形态学规则、表格规范化和动态形态形式生成抽出为前后端共享模块；词条读取 API 搜索现在会纳入自动生成的形态形式和词条级 override，并在一致性检查中覆盖。
- 词条读取 API 新增 `fields` 搜索字段参数，为后续手动配置搜索对象字段和例句升级为语料链接预留接口。
- 标签显示替换、词性识别、标签列表规范化、红色高亮标签和词条列表标签显示上限抽出为前后端共享模块，减少词条列表、搜索和 facets 之间的语义分叉。
- IPA 设置规范化、自动生成、规则上下文匹配、音节切分、重音解析和 IPA 对比工具抽出为前后端共享模块，为后续质量检查、数据分析和批量 IPA 逻辑后端化预留统一语义。
- 形态学规则引用解析、函数调用提取、替换语法校验和函数对象校验并入共享形态模块；后端保存形态配置时也会拒绝语法或函数对象错误。
- 词条搜索字段白名单、搜索字段文本抽取和读取 API 搜索文本生成抽出为前后端共享模块；前端本地筛选、搜索字段分析和后端词条查询现在复用同一套字段语义。
- 词条搜索的 fuzzy 评分与匹配逻辑并入共享搜索模块；读取 API 增加 `fuzzy`、`tagFuzzy` 和 `fuzzyFields`，为后续按字段分别配置模糊匹配预留接口。
- 质量检查的问题生成、Gloss 例句问题识别、词源网络问题识别、质量问题分组工具和词源来源解析抽出为共享质量模块，减少数据分析和高级筛选之间的语义分叉。
- 质量检查从数据分析子页迁移为独立工具导航页面；数据分析总览中的“质量问题”卡片现在跳转到质量检查页面，Glossed 例句分类只显示问题列表，不再额外统计 Glossed 例句数量。
- 词汇网络视图接入 `/entry-relations/:entryId` 读取 API；来源匹配和衍生关系优先使用后端结果，API 不可用时保留本地关系计算兜底。
- 词根模式新增 `/root-groups` 读取 API；前端词根模式正常路径改为直接使用后端词根组结果，不再每次本地计算完整分组做一致性校验，也不再在大词典 API 截断或失败时回退前端完整分组。API 返回分页截断结果时先渲染已返回页面，请求失败时显示失败状态，避免 3k/10k 测试词典把重型分组重新压回前端主线程。
- `NEXT_IMPLEMENTATION_HANDOFF.md` 和 `API_CONTRACT.md` 记录词根模式分页/窗口化 UI 暂缓到后端关系索引或 SQLite 评估之后再做，避免现阶段用简单无限滚动造成滚动条只能代表已加载页面的误导体验。
- 普通词条列表改为直接使用 `/entries` 查询 API 返回的顺序，不再每次先在前端本地筛选完整词条列表做一致性校验；本地筛选逻辑暂时保留给高级筛选、API 不可用和后续过渡路径。读取 API 临时上限提高到 10000，以避免未接入窗口化前截断当前 1k/10k 压测词典。
- 词性筛选选项改为直接使用 `/facets` 返回的词性集合，不再每次先在前端本地统计完整词性集合做一致性校验。
- 共享词条关系索引新增 `source key -> derived entries` 反向索引；词汇网络、词根模式、质量检查和 `/entries?derivedFrom=` 可复用同一关系索引，避免按来源查找衍生词时重复扫描全词条。
- 词根模式在 `/root-groups` 结果尚未完整加载时暂时禁用“展开全部”，避免大词典下该按钮重新触发前端完整词根分组。
- 数据分析总览移除“质量问题”卡片；数据分析渲染不再默认生成质量检查报告，质量检查仅由独立质量检查页面和质量高级筛选显式触发。
- 质量检查页面和质量高级筛选改用独立的质量报告构建入口，不再经过数据分析报告，也不再附带生成覆盖率、IPA、形态学、搜索字段和活动统计等无关分析内容。
- 词根模式、词汇网络和后端词源关系查询开始复用共享的词条关系模块；`API_CONTRACT.md` 补充后续词根模式读取 API 与关系索引优化设计。
- 数据分析页新增保守的报告缓存；词典内容、设置、语言、搜索或排序未变化时，页内切换标签会复用既有分析报告，避免重复全量重建。
- 质量检查页新增保守的报告缓存；词典内容、IPA 设置和界面语言未变化时，页内切换优先级/模块标签和刷新质量高级筛选会复用既有质量报告。
- 质量检查的词源来源解析和循环检测开始复用共享词条关系索引，减少来源检查时对全词条列表的重复扫描。
- 数据分析报告构建拆分为上下文、关系、覆盖率、标签、词形、IPA、形态学、搜索和活动等内部切片，并保留旧 report 输出结构；同时新增总览 widget 依赖声明，为后续可配置总览和按需分析 API 预留接口。
- 数据分析渲染开始按当前页面/子页构建必要切片；例如编辑进度页不再计算 IPA 或形态学统计，IPA 与形态学详情页也不再默认生成标签、词形和活动统计。
- 数据分析缓存从页面 report 缓存升级为切片缓存，不同页面和子页可复用相同的 IPA、覆盖率、关系、标签、词形、形态学、搜索或活动统计切片。

- 语料单元编辑内容现在可以即时更新渲染预览，不必等待保存完成。
- 自动保存不再退出当前语料编辑状态，也不会清除当前草稿和撤销历史。
- 数据分析与其他设置卡片改为响应式瀑布流布局，长卡片不再撑开同一网格行。
- 自动 IPA、自动形态学和其他设置移除不必要的最外层大卡片，使子区域边界更清晰。
- 其他设置改进宽屏空间利用，并调整渲染对象、标签替换等长文本输入区域。
- README 补充语料库、Gloss 样式、保存策略、导入保护与全局语言设置说明。
- IPA 映射改为严格按照列表从上到下匹配；规则始终读取原始词形，生成结果不会被后续规则再次读取，较早规则消耗的字符不会再参与后续匹配。
- 改进中间宽度与窄屏下的通用布局收缩和换行；词条详情、设置、分析等卡片不再被内部 Grid、Flex 或长文本撑出边界。
- 形态表格与对齐 Gloss 在空间不足时保持卡片内部局部横向滚动，不再引起页面级横向滚动。

### 性能

- 全局渲染改为只更新当前打开的功能模块；未打开数据分析时不再生成分析报告。
- 词条列表、词根模式衍生词和语料列表加入动态高度虚拟滚动，页面只保留可见区域附近的卡片。
- 虚拟列表保留选中项跳转、词根展开、多来源上下文与窄屏独立滚动行为。

### 修复

- 修复语料库为空时仍存在额外滚动空间的问题。
- 修复 Gloss 多行对齐未生效、`\ft` 参与对齐以及对齐后字号异常缩小的问题。
- 修复语料自动保存导致编辑状态、草稿和撤销记录丢失的问题。
- 修复词根模式下跳转到衍生词时可能定位到错误实例的问题。
- 修复设置和分析网格因超长卡片产生大面积垂直空白的问题。
- 修复原生单行输入框裁切 `d͡ʒ`、`t͡ʃ` 等跨字符组合附加符号的问题；IPA 映射、音节规则、沙盒输入以及完整和局部词条发音现在使用保持单行行为的文本区域。
- 修复特定窗口比例下词条详情子卡片和操作区超出父卡片，以及最窄视口仍出现页面横向滚动的问题。

### 安全与兼容性

- 词条、语料块、语料层和语料单元使用跨类型实体 ID 唯一性检查，前后端保存时都会阻止重复 ID。
- 保持旧词典 JSON 的配置默认值与数据结构迁移逻辑，新增设置缺失时会使用兼容默认值。
- `data/` 继续保持 Git 忽略，个人词典不会进入版本控制。
