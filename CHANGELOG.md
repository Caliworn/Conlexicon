# Changelog

本文件记录 Conlexicon 的重要功能、行为与兼容性变更。

## New

### 改进

- 高级筛选查询化 F1 新增共享、可序列化的 `EntryQuery/EntryFilter` 模型：筛选、自由文本搜索、排序、DTO 形状和分页窗口现在具有明确边界，现有 `/entries` 参数会先归一化为同一结构；查询会话 descriptor、cursor digest 和缓存键直接使用同一规范化身份，不再各自拼装平铺字段。模型已为字段存在性、来源数量和活动日期条件定义严格校验，但在 F2 SQL 编译完成前会明确拒绝这些尚未接线的条件。
- 将 API、SQLite、查询缓存和高级筛选等技术文档整理到 `docs/`，新增文档索引并修正全部当前文档间的相对路径；根目录继续保留项目 README、Changelog、长期协作规则和阶段性交接入口。
- 全面复核 README、交接、API、SQLite、迁移与查询缓存文档的实现状态：将查询会话 Q1–Q4、词根搜索/定位、词汇关系 API、SWR 和局部词条渲染从待办改为已完成，修正当前 schema、错误码、JSON 定位和中等宽度导航说明，并保留高级筛选、分析/质量 API 与语料升级等真实待办。
- 完成高级筛选查询化 F0 清点：区分可直接进入 EntryQuery 的稳定条件、需要独立服务端结果会话的分析/质量结果，以及不属于筛选的导航动作；记录全部现有入口、刷新失效和语义债务，为后续移除前端完整 ID 数组与本地快照扫描确定边界。
- 正式移除旧 JSON runtime repository 与 `CONLEXICON_REPOSITORY` 后端切换入口，主服务固定使用 SQLite；旧 JSON 单文件导入、导出和目录迁移仍由独立 conversion/migration 边界处理，批量迁移脚本改为直接只读旧数据目录。
- 重整存储测试边界：删除 JSON repository 完整契约入口，将共享 repository contract 收紧为 SQLite 正式契约，并把模型与旧 JSON 转换检查拆为独立脚本；服务启动 smoke 现在只验证 SQLite 数据文件。
- 删除 repository contract 中已被逐值搜索 record 专项检查覆盖的旧标签扁平数组断言，避免重复绑定内部投射形状。
- 清理正式运行路径中已移除实体 ID 的代码残留：IPA 文本规则不再传递无效的 ID 预留参数或执行空的 scoped ID 校验，词条形态组和词典设置规范化也不再保留旧 ID options；旧 JSON 字段剥离仍集中由导入 migration 负责。
- “备注”搜索字段现同时覆盖词条备注、释义备注和词条形态组备注；不同形态组的备注仍按独立值匹配，词条列表的命中摘要会分别标明三种备注来源。该功能复用现有 SQLite 搜索投射，无需调整 schema 或加入旧字段兼容。
- 词汇网络第一版升级为分层 SVG 关系图：来源、当前焦点和直接衍生词按稳定节点 ID 布局，点击关联词条时复用已有节点并平滑重新聚焦、同步更新有向连线，不再执行固定 160ms 的整面板淡出重绘。关系未缓存时先保留可确定的相邻关系，响应返回后补入新节点；窄屏自动切换为纵向关系图，并补充节点计数、应用内悬浮详情、键盘激活、弹层焦点与 reduced-motion 支持。
- 新增并接通普通词条与词根模式的后端目标窗口定位：自动跳转会按当前查询直接装入包含目标的窗口，不再扫描完整活动词典推算页号；词根模式会先定位父级窗口，再读取整组衍生词，并支持指定多来源词条的目标根上下文。
- 窗口化查询正式区分顺序分页 `nextCursor` 与随机访问 `windowCursor`；普通词条和词根组响应在每个窗口都返回后者，前端不再借用可能在末页为空的“下一页”游标来加载远端窗口，为后端目标定位 API 提供稳定契约。
- 词条列表查询与按需详情读取接入两段式 stale-while-revalidate：有已完成内容时，读取开始后的 200ms 内保持原界面，超过后用统一的“正在更新”遮罩使旧列表或详情变淡；旧详情从请求开始即进入 `inert`，避免遮罩出现前或通过键盘继续操作旧词条。首次加载仍直接显示加载状态，请求失败则进入明确错误状态，不回滚或自动重试。滚动补载后续数据窗口继续使用局部占位，不触发整栏遮罩。
- 词根模式“全部展开”改为跨父级窗口的全局状态，不再要求全部窗口同时驻留；父级首窗提供轻量窗口统计以估算完整展开高度，组内内容仍按可见范围加载。全展开状态下可单独收起词根作为例外，父级窗口淘汰不会丢失该意图；全部展开或全部收起会清理上一轮例外。
- 查询 cursor 升级为绑定服务进程、词典缓存 generation 和查询 descriptor 的不透明版本化游标；非零远端窗口必须携带有效 cursor，写入、重启或查询条件变化会返回可区分原因的结构化失效错误，缓存淘汰则透明重算。
- 更新 handoff、SQLite 后端/迁移计划、API 契约和 README，移除搜索 S4、JSON 完整契约与早期目录拆分等过时规划，并记录查询缓存、数据窗口化、按需详情和关系拓扑已完成后的剩余查询消费者路线。
- 新增查询会话与结果缓存设计，明确前端紧凑查询缓存、后端运行时会话、查询身份、词典级失效、容量淘汰和可重建 cursor；缓存淘汰只触发重算，不影响分页正确性，并将列表 DTO、按需详情和词根组子项拆分提前到数据窗口化之前。

### 性能

- 词条详情的来源按钮、衍生词列表、完整编辑中的只读衍生词和词汇网络现在共用 `/entry-relations/:entryId` 及前端关系缓存；切换词条不再为每个来源和衍生词区域分别扫描、排序完整词典并重建关系索引。SQLite 来源响应同时返回匹配词条 summary，词汇网络也不再为了补全来源卡片遍历活动词典。
- 词条切换不再调用全局 `render()`：现在只更新已渲染卡片的选中态、按需词条详情、词汇网络可见状态和必要的列表高度/自动滚动，不再重建导航、筛选器、查询窗口、虚拟列表 DOM、卡片高度与 offsets。词汇网络关闭后跳转到当前网络节点也复用同一局部提交路径。
- 目标定位复用现有查询结构：严格词条查询在 SQLite 内计算排序位置，fuzzy 词条与词根组分别通过会话的反向下标直接定位；返回目标窗口时不需要前端加载或扫描完整活动词典。
- fuzzy 词条会话和词根组会话分别为已有有序结果建立 `entryId → resultIndex`、`rootId → resultIndex` 索引；索引随会话一次生成并受同一容量估算和 LRU 管理，为目标定位 API 提供 O(1) 的结果下标读取，不额外扫描 SQLite。
- 稳定词根拓扑新增 `entryId → rootIds` 和 `rootId → group` 反向索引，词根组子项读取不再线性扫描查询会话，词汇网络关系读取也不再为每次请求递归遍历祖先和后代；定位成本接近 O(1)，仅保留与实际返回组大小相关的 DTO 构建成本。
- 词根模式新增独立 relation generation 的稳定拓扑缓存：词根与全部衍生词的 ID 关系不再随搜索词、搜索字段、排序或无关保存重复构建；搜索查询直接在 SQLite 静态/形态投射中筛出命中 ID，再对缓存拓扑生成查询视图，不再导出完整词典 snapshot。词典摘要的词根计数也复用同一拓扑。只有词条增删、lemma/来源变化和整库替换会使拓扑失效；普通词条保存与批量 patch 仅更新轻量排序记录，其他模块保存完全保留拓扑。
- 查询缓存 Q4 将普通词条和父级词根组列表改为纯滚动数据窗口：分别按 200/100 的小窗口读取，各自最多保留 5 个已加载窗口；未加载页以完整结果总数建立等高占位，原生滚动条从首屏起即可代表完整列表，远端页按可见范围预取并按距离淘汰。单个词根组展开后一次读取整组衍生词，不再维护嵌套窗口、cursor 或单组硬上限。
- 窗口替换会继承占位页的高度比例，并在非滚动状态恢复现有虚拟列表锚点；自动跳转会先补载目标所在窗口，避免加载或淘汰远端页时让当前视口无故跳动。
- 查询缓存 Q3 将普通词条列表切换为直接消费 SQLite summary DTO，并以最多 12 项的按需详情缓存读取当前词条；列表不再把查询结果压成 ID 后回到完整词典快照取卡片数据，多义项预览与搜索命中位置仍保留。
- 词根模式的折叠组响应改为只返回词根 summary、衍生词数量和匹配数量；组内衍生词在展开后通过独立端点一次读取整组，避免折叠状态预载无关子项。
- 查询缓存 Q1 新增前端紧凑 LRU：词条查询、词根组和词性 facets 会复用成功结果并合并同 key 请求，缓存受 4 项与约 16 MiB 双重限制；保存、覆盖或删除词典后按词典失效，错误和过期代结果不会进入当前 UI。
- facets 查询键改用词典 `updatedAt`，不再为生成缓存签名扫描全部词条标签。
- 搜索输入防抖由 150ms 调整为 250ms，减少连续输入期间产生的中间查询，同时保持停止输入后的及时反馈。
- 查询缓存 Q2 新增后端运行时 `QuerySessionCache`：fuzzy 词条查询复用有序命中 ID，词根模式复用根/衍生关系 ID，页面内容仍从 SQLite 按需组装；相同查询合并构建，并受每词典 8 项、全局约 64 MiB、2 分钟 idle TTL 和 LRU 限制。
- 所有 SQLite 词典写入只在成功后递增运行时缓存 generation 并按词典失效；空 patch 和失败写入保留有效会话。查询命中路径同时改用轻量 metadata＋settings 上下文，不再读取无关 docs、corpus 或完整形态配置。
- 新增只读 Q2 查询会话基准脚本，可直接针对现有 SQLite 测试目录比较 fuzzy entries 与 root groups 的首次冷查询和重复热查询。

### 修复

- 快速连续切换词条时，较早完成的详情请求不再覆盖后一次选择的自动滚动目标或关闭抽屉。
- 词条列表卡片的局部选中态同步补充 `aria-current`，切换词条和词汇网络返回后，视觉选中态与辅助技术读到的当前位置保持一致。
- 修复查询切换的 SWR 旧列表可能提前“完成”自动滚动、随后被新首窗替换而丢失目标的问题；待定位状态现在只允许由当前查询的就绪窗口完成。
- 为 Gloss 字体、字号、样式控件和逐字段搜索开关补充唯一表单字段名称，消除 Chromium/Edge 对缺少 `id` 或 `name` 的表单控件诊断，同时保持既有 `data-*` 保存逻辑不变。

## 2026-07-16

### 改进

- 搜索 S3.1 建立统一的逐值检索 record 契约：共享搜索模型现在为每个词形、标签、义项、备注和词源值保留字段及来源定位，现有 JavaScript matcher 也复用同一边界；SQLite 静态 projection 与 API `searchHits` 的后续结构和失效范围已同步写入契约文档。
- 搜索 S3.3 接通 SQLite 逐值 projection 与词条列表：静态非模糊搜索会返回实际命中的 `searchHits`，搜索摘要据此保留义项定位，不再为静态结果重复遍历本地字段或生成动态形态。

### 性能

- 搜索 S3.2 新增无独立 ID 的 SQLite `entry_search_values` 派生表；导入、词条保存、批量 patch、删除以及相关搜索规范化/标签替换设置变化会事务性维护词形、IPA、标签、释义、例句、备注和词源 records。动态形态不写入该表。
- 搜索 S3.3 将全部非模糊静态字段查询切换到 `entry_search_values`，覆盖 Unicode、NFC、case folding 和自定义等价规则，避免读取完整词典 snapshot。
- 搜索 S4 新增独立、无实体 ID 的 `entry_morphology_search_values` 派生表，保存形态求值顺序、真实模板组/子表/单元格坐标及原始/规范化值；词条写入局部维护，形态配置或搜索规范化变化全量重建，删除外键级联。严格及 fuzzy 的形态单字段、静态字段和混合查询现在均直接读取两张 projection，并返回统一 `searchHits`，不再导出完整词典或逐词条动态生成形态。SQLite 连接通过确定性函数复用共享 fuzzy 评分；大型候选集顺序扫描 projection，避免大型 `IN (...)` 查询和全部 records 的 Node.js 物化。
- 词条查询的结构筛选和无搜索路径不再为 summary 响应重建完整词条；前端查询失效键改用词典 `updatedAt`，搜索输入增加 150ms 防抖，避免每次按键都生成大型词典内容签名并立即发起读取。

### 修复

- SQLite 词条排序增加唯一 ID 作为最终稳定排序键，避免同词形、同时间及同位置记录在分页边界间漂移；普通词条列表和词根模式在 API 结果被 `hasMore` 截断时会在列表末尾显示最小加载数量提示。
- 补齐词条编辑、词条详情、筛选排序控件及各主要页面区域的无障碍名称本地化；切换中英文后，屏幕阅读器不再继续读出硬编码的中文标签。

## 2026-07-14

### 改进

- IPA 映射规则移除持久化实体 ID，改为按顺序保存的纯文本规则；词条形态组移除 `emorph` UUID，以 `entry_id + template_group_id` 作为组合身份和 override 外键。两类旧 ID 只在 JSON 导入迁移时删除，不再参与运行期跨类型 ID 防撞。
- 自由文本搜索现可按词典配置 NFC、Unicode 17 默认大小写折叠及自定义字面等价规则；规则按最长变体优先、单次替换，空规则和冲突规则会在保存前给出本地化提示。严格模式为默认值。
- 词条列表、词根模式、搜索摘要与高亮、搜索字段分析和词源自动补全共用同一搜索 normalizer；结构键与词源关系键继续保持各自既有的独立匹配边界。
- 标签筛选、手动词性识别、红色高亮标签和自动形态模板匹配改为大小写与码位精确匹配；SQLite `entry_tags` 不再保存 `normalized_tag` 派生列。词源 `source_key` 暂保持原有宽松匹配，之后改为稳定词条 ID 引用时再单独整理。
- 开发期 SQLite 移除没有实际升级脚本的 schema 版本字段与 migration 表；当前 schema 不匹配的测试库继续从 JSON 重建。下一个正式 release 才重新引入版本化 schema migration。
- 导入迁移现在会完成并清理旧词条、设置和语料字段；前端、核心模型与 SQLite 写入侧只按当前数据结构运行。前端启动也不再接受 `/api/state` 中的完整词典 payload，形态规则错误标签也不再识别旧单元格坐标格式。
- 其他设置的搜索配置改为按词形、IPA、标签、释义、例句、备注、词源和形态学分别保存“参与搜索 / 模糊匹配”开关；关闭字段搜索会禁用但保留其模糊匹配偏好，且不允许保存全部字段均关闭的配置。词源自动补全的模糊匹配也迁入同一 `search` 设置结构。
- 词条列表、词根模式、前端本地筛选、搜索命中摘要和数据分析的“当前搜索命中字段”现在统一读取词典级搜索字段配置；关闭形态搜索时不再生成形态搜索文本。读取 API 同步移除旧 `fuzzy` / `tagFuzzy` 参数，仅接受字段级 `fuzzyFields`。
- 基础自由文本搜索改为逐个独立值匹配：释义、标签、来源、备注和例句不再因彼此拼接而意外命中，字段级模糊匹配也遵守同一边界；需要多个标签或条件组合时应使用高级筛选。
- 词条列表搜索结果不再重复渲染首条释义；命中后续义项时保留命中摘要，并显示原义项序号（如“释义2”）。
- 精确搜索高亮现在会标出同一词形、释义、标签或命中摘要值中的全部非重叠命中；NFC、大小写折叠或自定义替换改变文本长度时，也会映射回正确的原始字符范围。

### 性能

- 词条读取 API 在搜索范围不含形态学、所有字段均为严格匹配、未启用大小写折叠或自定义规则且查询为可打印 ASCII 文本时，直接在 SQLite 中筛选静态字段；其他配置在 S3 检索投影完成前使用共享 JavaScript matcher。
- SQLite 的词典元数据、其他设置、IPA 设置、语言文档和语料保存不再在 repository 内部重建完整词典 snapshot，改为直接返回各自的局部 payload。
- SQLite 批量词条 patch 保存改为直接返回本次更新的词条及可选设置，不再为 repository 返回值重建完整词典 snapshot。
- SQLite metadata 读取改为轻量路径；autosave 现在拒绝缺少 docs/corpus 的无效请求，同一请求包含文档和语料时会正确合并两个局部结果。
- 词汇网络移除前端本地关系索引的双算与大型词典全量签名；改为直接等待 SQLite 关系 API，并使用词典更新时间作为轻量缓存键。
- 编辑表单现在按规范化后的语义内容判断是否有未保存更改：未改动的完整/局部词条编辑离开时不再触发保存确认；手动保存未改动的词条、词典配置、其他设置、IPA、形态学、语言文档或语料库时不再发送写入请求，并提示没有需要保存的更改。

### 修复
